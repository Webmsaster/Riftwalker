## Wichtige Erkenntnisse
- **Cache-Layer-Surprise #2**: Genau wie Bug #45 (procedural SFX bypassed by stale .wav cache), hier Bug-Klasse "deploy von stalem Build-Snapshot statt Repo". Pattern: jede File-Pipeline mit Zwischenstufe (build/deploy/cache) kann silentlich alte Daten ausliefern, wenn die Generation außerhalb der Pipeline läuft (`--dump-sfx`, `gen_music_v3.py`).
- **SDL Audio-Device-Default ≠ Hörbar**: Windows-Default-Audio-Device kann ein HDMI-Monitor ohne Speaker sein. `Mix_OpenAudio(NULL)` spielt dann in den Void. Fix: Device-Liste enumerieren und nach Headset/Speaker-Pattern filtern.
- **pyfluidsynth hardcoded DLL path**: Erwartet `C:\tools\fluidsynth\bin` — wenn die DLLs woanders liegen, fluidsynth import crasht mit `FileNotFoundError`. Genau dort hin entpacken.
- **One-time-per-run Bonuses**: `applyAscensionModifiers()` wird in `generateLevel()` per Floor aufgerufen. Alles was nur einmal pro Run feuern soll → gate hinter `!m_ascensionModifiersApplied`, flag in `startNewRun()` resetten. Selbe Logik wie startShardBonus (Bug #46).
